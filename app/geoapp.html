Layers

BasemapSelect background map from the list
Add web mapAdd WMS, WFS, WMTS/XYZ or ArcGIS MapServer by URL. WMS overlay: single GetMap image per viewport (non-tiled), OL reprojects from server native CRS (EPSG:4258/4326/3857) to map CRS — ideal for raster overlays. WFS: vector features (polygons/points) fetched per viewport above the configured min zoom (default 15 — lower values risk huge downloads), rendered as GeoJSON — best for cadastral/topographic data. Tap "Get layers" to auto-detect layer names. Overlay layers are saved and restored on next launch with their opacity and visibility state.
WFS filterFilter attribute and filter values fields let you show only matching features. Load the layer without filter first, tap a feature to see attribute names, then re-add with the filter set. Multiple values: comma-separated (e.g. A001,B002). Uses CQL_FILTER — supported by GeoServer-based services (most Italian WFS). If no features appear with filter active, check the attribute name and values.
WFS selectionWhen WFS layers are loaded a badge with "⊙ Select" appears. Tap it to activate click-to-select mode: if multiple WFS layers are loaded a picker appears to choose which layer to target — only features of that layer respond to tap-to-select, the others open normally. Once a layer is chosen, tap any feature to select/deselect it instantly (no popup). The badge shows the active layer name. Tap "⊙ Select" again to reset. The popup "☆ Select" button also works as an alternative. The count badge shows selections — tap ⬇ KML to export, ✕ to clear all.
Load fileImport KML, KMZ, GeoJSON, GPX — drag & drop or tap. KML exported from WFS selection shows catasto attributes in the popup when tapped.
Layer listToggle visibility, adjust opacity, reorder by drag, export or delete. Double-tap the layer name to rename it.
Hollow ∅Tap the ∅ button on any loaded file layer to switch it to hollow mode — only the border is drawn, fill is transparent. Tap again to restore. Works on KML, GeoJSON and GPX polygon layers. Hollow state is saved and restored on next launch.
Edit KML ✏Tap the pencil icon on a KML layer to enter vertex edit mode — drag vertices to move them, double-tap a vertex to delete it, drag a dashed midpoint marker to add a new vertex. Save commits changes back to the layer and updates the stored KML. Cancel discards all edits.

Drawing

Toolbar ↖Draw marker, polyline, polygon, rectangle, circle — top-left map toolbar
Edit toolbar ↖Move and reshape existing shapes — drag vertices to reposition, double-tap a vertex to delete it, drag a dashed midpoint marker to add a new vertex. Save commits changes, Cancel discards them
Tap shapeName it, add a description, change icon/color/opacity, export as KML, or delete. Name and description are auto-saved as you type — no need to confirm.
Auto-saveDrawings (including name, description, color, icon) are saved automatically after each change and restored on next open
Export allExport all drawings as KML or GeoJSON (includes color, icon, type, opacity)
Clear allPrompts confirmation before deleting all drawings
Dissolve KMLIn the KML Tools section of the Draw panel — tap "Dissolve KML features" to open a layer picker. Select one or more loaded KML layers with checkboxes, then tap "Dissolve selected": all polygon features from the chosen layers are merged into a single geometry, internal boundaries removed. Holes and non-contiguous areas are preserved. Enter name and description, then save as a new KML layer. Requires turf.min.js in the app folder.

GPS & Track

GPS btn ↘Center map on position — tap marker for full coords + elevation
Screen onScreen wake lock acquired automatically while GPS is active — screen stays on
Draw › GPSStart/stop GPS track — when done, export as GPX or KML, and optionally save the track as a drawing (it will appear in the drawings list with the name you choose)
